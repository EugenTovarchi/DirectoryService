# AuthService EF Migration Bundle

EF Core migration bundle is a self-contained executable generated by `dotnet ef migrations bundle`.
It can apply AuthService EF Core migrations to an AuthService database without running the Web
application container.

AuthService has an initial migration, `InitialCreate`, which creates the `auth_users` table.

## Build Bundle

Run from repository root:

```powershell
.\scripts\build-authservice-efbundle.ps1
```

The script runs:

```powershell
dotnet ef migrations bundle `
  --project backend/AuthService/AuthService.Infrastructure.Postgres/AuthService.Infrastructure.Postgres.csproj `
  --startup-project backend/AuthService/AuthService.Web/AuthService.Web.csproj `
  --context AuthServiceDbContext `
  --configuration Release `
  --output artifacts/migrations/AuthService/efbundle.exe `
  --force
```

The output artifact is:

```text
artifacts/migrations/AuthService/efbundle.exe
```

The `artifacts/` directory is ignored by git.

## Run Bundle

Pass the connection string at runtime:

```powershell
powershell -ExecutionPolicy Bypass -File .\scripts\run-authservice-efbundle.ps1 -ConnectionString "Host=localhost;Port=5434;Database=auth_service;Username=postgresUser;Password=postgresPassword"
```

Do not commit real production secrets. Use environment-specific secret storage, CI/CD secrets, or a
secret manager for production connection strings.

The `auth_service` database must already exist before running the bundle. EF migrations create schema
objects inside the database; provisioning the database itself is infrastructure responsibility.

For local Docker development, create the database manually if it is missing:

```powershell
docker exec directory-service-postgres createdb -U postgresUser auth_service
```

## Docker Compose Migrator

AuthService uses one Dockerfile with two final targets:

- `app`: the AuthService.Web runtime image. It runs `AuthService.Web.dll` and does not contain the EF
  bundle or migration scripts.
- `migrator`: the one-shot migration image. It contains the EF bundle and `migrate.sh`, runs the
  bundle once, and exits.

Build the migrator image:

```powershell
docker compose -f docker-compose-dev.yml build auth-service-migrator
```

Run the migrator:

```powershell
docker compose -f docker-compose-dev.yml run --rm auth-service-migrator
```

The migrator receives `ConnectionStrings__DefaultConnection` from `AuthService.Development.env`.
It does not create `auth_service`; create the database first if needed.

## Deployment Note

In a production-like deployment, run the AuthService migration bundle before deploying or starting the
new AuthService container version:

1. Provision the database through infrastructure.
2. Run the `migrator` image with the target environment connection string from secrets.
3. Roll out the `app` image after the migrator succeeds.

This keeps database schema changes explicit and separate from the application startup path.
